Xinli Portal Server Console - Searching Help

Xinli Portal Server supports searching for
  • Activity
  • Certificate
  • Nas
  • Session

Overview

Searching is implemented base on JPQL (java persistence query language). Searching syntax is the same as JPQL. You can create query using 'and', 'or', sub query and etc., and you can also use 'like' to create queries, i.e. activity.action like '%SESSION'. Results can be sorted by 'order by' clause.

Search Activity
keyword values
activity.facility SYSTEM, SERVER, ADMIN, PORTAL
activity.severity FATAL, ERROR, WARN, INFO, DEBUG, TRACE
activity.source text expression
activity.action AUTHENTICATE, CREATE_SESSION, GET_SESSION, UPDATE_SESSION, FIND_SESSION, DELETE_SESSION
activity.result text expression
created Date expression, i.e. activity.created before '2016-5-5'
Search Certificate
keyword values
certificate.appId text expression
certificate.vendor text expression, i.e. certificate.vendor = 'xinli'
certificate.os text expression, i.e. certificate.os like '%ios'
certificate.version text expression
certificate.disable true or false
Search Nas
keyword values
nas.name text expression
nas.ipv4Address ipv4 text expression, i.e. nas.ipv4Address = '192.168.1.1'
nas.ipv6Address ipv6 text expression
Search Session
keyword values
session.nas.name text expression, same as nas.name
session.nas.ipv4Address ipv4 address expression
session.nas.ipv6Address ipv6 address expression
session.credentials.username username expression
session.credentials.ip ip address expression
session.credentials.mac mac address expression(lowercase, separated by ':'), i.e. session.credentails.mac = '40:8d:5c:1e:ce:3e'
session.certificate.appId text expression
session.certificate.vendor text expression
session.certificate.os text expression
session.certificate.version text expression
session.startTime Date expression, i.e. session.startTime before '2016-5-5'
